# 변경 내역

## 2026-04-16

### 주요 기능 추가

- DeepL API를 이용한 한국어 기준 번역 기능을 추가했습니다.
- 언어 탭에서 현재 선택한 언어로 번역할 수 있습니다.
- 번역 시 기본 4개 섹션 제목은 API 번역 대상에서 제외하고, 언어별 고정 문구를 사용합니다.
- DeepL 월 사용량과 번역 예상 글자 수를 화면에서 확인할 수 있습니다.
- 번역 결과는 적용 후 자동 저장됩니다.

### 자동 저장 개선

- 60초 주기 자동 저장을 추가했습니다.
- 프로젝트 이동 또는 새 프로젝트 생성 전에 현재 작업 중인 프로젝트를 먼저 저장합니다.
- 자동 저장은 저장 시작 시점의 스냅샷을 기준으로 처리하도록 변경했습니다.
- 프로젝트 A 수정 중 프로젝트 B를 선택할 때 A 데이터가 B에 덮어써지는 문제를 수정했습니다.
- 프로젝트 복원 중에는 입력값 동기화가 실행되지 않도록 막아, 불러오는 중 이전 프로젝트 데이터가 섞이지 않게 했습니다.

### 편의 기능 개선

- 저장 상태 표시를 추가했습니다.
- 저장되지 않은 변경사항이 있을 때 브라우저 종료/새로고침 경고를 표시합니다.
- 새 프로젝트 생성 시 저장 버튼을 누르지 않아도 목록에 즉시 추가됩니다.
- 이미지 다중 업로드와 드래그 앤 드롭 업로드를 지원합니다.
- 다운로드 이미지 품질 선택 기능을 추가했습니다.
- 입력 플레이스홀더 색상을 더 연하게 조정했습니다.

### 배포 및 보안

- `.env`를 Git 추적 대상에서 제외했습니다.
- 운영 서버에서는 `.env` 파일을 직접 생성하거나 복사해야 합니다.
- IIS에서 `.env` 접근을 차단하도록 `web.config`를 보강했습니다.
- 배포 가이드에 DeepL API 키 설정 내용을 추가했습니다.

### 운영 배포 전 확인 사항

- 운영 반영 전 `database.db`를 백업하세요.
- 업로드 이미지를 유지하려면 `uploads/` 폴더도 함께 백업하세요.
- SQLite WAL 파일이 있는 경우 서버를 중지한 뒤 백업하는 것을 권장합니다.
- 운영 서버에 `.env` 파일과 `DEEPL_API_KEY` 값이 있는지 확인하세요.
- 배포 후 기존 프로젝트 열기, 프로젝트 이동, 새 프로젝트 생성, 번역 1회 테스트를 권장합니다.
